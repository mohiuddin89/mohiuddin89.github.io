Debugging

Production problems I'm comfortable investigating

How I think when something breaks — signal, what I check, likely path.

CrashLoopBackOff

Check: container logs and events, exit code, recent image or config change.

Path: bad config, failing probe, or missing dependency — fix or roll back the release.

OOMKilled (exit 137)

Check: memory requests/limits, recent traffic, whether usage grows over time.

Path: right-size limits, or fix the leak if memory only climbs.

ArgoCD OutOfSync / Degraded

Check: diff between Git and cluster, sync hooks, RBAC, image tag.

Path: reconcile the manifest and fix the drift, or roll back the offending commit.

AWS AccessDenied

Check: IAM policy, role trust, the exact action and resource in the error.

Path: grant the least privilege that actually works — no broad wildcards.

Failed rollout

Check: events, readiness probes, resource pressure, image pull status.

Path: roll back to the last good Helm release, then fix forward.

Red pipeline blocking a deploy

Check: failing stage logs — test, build, or scan — and what changed last.

Path: fix the failing step or flaky dependency and get it green again.
